problem etc... #

YAML is a JSON superset in more ways than syntax alone. Among other things, it supports custom data types, via local / application-specific tags.

Say you need to read (or write) an AWS Cloud Formation template:

EC2Instance:
  Type: AWS::EC2::Instance
  Properties:
    ImageId: !FindInMap [
      AWSRegionArch2AMI,
      !Ref 'AWS::Region',
      !FindInMap [AWSInstanceType2Arch, !Ref InstanceType, Arch],
    ]
    InstanceType: !Ref InstanceType

Or you need to safely read untrusted YAML containing native Python objects:

!!python/object/new:module.Class { attribute: value }

Warning

Historically, yaml.load(stream) was unsafe for untrusted data, because it allowed running arbitrary code. Consider using use safe_load() instead.

Details.

For example, you could do this:

>>> yaml.load("!!python/object/new:os.system [echo exploit!]")
exploit!
0

There were a number of CVEs about it.

Since PyYAML 6.0, yaml.load() requires an explicit Loader argument.

Also, 5.* added two new functions and corresponding loaders:

  • full_load() resolves all tags except those known to be unsafe (note that this was broken before 5.4, and thus vulnerable)
  • unsafe_load() resolves all tags, even those known to be unsafe (the original load() behavior)

safe_load() resolves only basic YAML tags, and remains the safest.

The YAML spec says:

In a given processing environment, there need not be an available native type corresponding to a given tag. If a node’s tag is unavailable, a YAML processor will not be able to construct a native data structure for it. In this case, a complete representation may still be composed and an application may wish to use this representation directly.

... doing the first for each tag is not possible, since we don't necessarily know ahead of time what we'll get.

... doing the second is possible, but what we get out of the box is unnecessarily verbose:

>>> text = """\
... one: !myscalar string
... two: !mysequence [1, 2]
... """
>>> yaml.compose(text)
MappingNode(
    tag='tag:yaml.org,2002:map',
    value=[
        (
            ScalarNode(tag='tag:yaml.org,2002:str', value='one'),
            ScalarNode(tag='!myscalar', value='string'),
        ),
        (
            ScalarNode(tag='tag:yaml.org,2002:str', value='two'),
            SequenceNode(
                tag='!mysequence',
                value=[
                    ScalarNode(tag='tag:yaml.org,2002:int', value='1'),
                    ScalarNode(tag='tag:yaml.org,2002:int', value='2'),
                ],
            ),
        ),
    ],
)
>>> print(yaml.serialize(_))
one: !myscalar 'string'
two: !mysequence [1, 2]

while the pyyaml follows the yaml spec:

That said, tag resolution is specific to the application. YAML processors should therefore provide a mechanism allowing the application to override and expand these default tag resolution rules.

Mind you, it's not complicated because pyyaml is complicated, but because yaml itself is; here's all the steps to load yaml:

There should be an easy way to parse and emit this kind of YAML without having to know/wrap each tag.

One way of doing it is to convert everything to almost-native types, but keep the tag for things with custom ones.

And this is what we're gonna do!

from yaml import SafeDumper, SafeLoader


class Loader(SafeLoader):
    pass

class Dumper(SafeDumper):
    pass

pyyaml is weird; the multiple roles above are provided by methods provided multiple parent classes – Constructor, Representer etc.

the Loader/Dumper subclasses all the required classes

however, extensibility does not happen through subclasses; each Loader/Dumper has a class-level registry of constructors, representers etc. ... but, they are bit method-like, in that they receive the Loader/Dumper as the first argument

on first use, it gets copied from the next parent class onto the child; we can't merge functionality by subclassing multiple Loader subclasses, because we'd have to merge the registries of the multiple subclasses by hands

class Tagged(wrapt.ObjectProxy):

    # needed to set the attribute on the proxy and not the wrapped object
    tag = None

    def __init__(self, wrapped, tag):
        super().__init__(wrapped)
        self.tag = tag

    def __repr__(self):
        return f"{type(self).__name__}({self.__wrapped__!r}, {self.tag!r})"

the proxy behaves identical to the proxied object (including things like isinstance()!), with two differences:

  • the proxy has a tag attribute
  • the repr()

def construct_undefined(self, node):
    if isinstance(node, ScalarNode):
        value = self.construct_scalar(node)
    elif isinstance(node, SequenceNode):
        value = self.construct_sequence(node)
    elif isinstance(node, MappingNode):
        value = self.construct_mapping(node)
    else:
        assert False, f"unexpected node: {node!r}"
    return Tagged(value, node.tag)

Loader.add_constructor(None, construct_undefined)

def construct_mapping(self, node):
    value = self.construct_pairs(node)
    try:
        return dict(value)
    except TypeError:
        return Pairs(value)

Loader.construct_mapping = construct_mapping
Loader.add_constructor('tag:yaml.org,2002:map', Loader.construct_mapping)

We also set Loader.construct_mapping so construct_undefined can take advantage of it too. It's a method, we could've put it in the Loader class definition, but we'll keep the style.
